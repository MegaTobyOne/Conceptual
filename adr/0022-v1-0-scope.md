# 0022 — v1.0 scope for initial assurance user testing

- Status: accepted
- Date: 2026-05-11

## Context

The v0.4 slice (UI resilience plus readiness hardening) closed all v0.4 quality gates and produced a publishable Explorer bundle. The owner now wants a **testable v1.0** that an Australian Government assurance user can drive end-to-end on a current MacBook Air, without third-party accessibility audit, chart image export, or local-authoring of Explorer artefacts.

ADR 0014 (v0.1 thin slice) pinned the operator spine: initialise workspace → author evidence-backed assessment data → snapshot → export → view in Explorer → copy posture brief. ADR 0016 (Australian context amplified) and ADR 0017 (ISM integration roadmap) added Home Affairs context and the read-only ISM source library plus Phase 2 mapping. v0.5 introduces Directions overlay and Action Impact ranking (see ADR 0023).

Several v0.2+ candidates remain deferred or undecided. This ADR pins what is in and what is explicitly out of v1.0, so the operator-validation loop can converge without scope drift.

## Decision

Ship v1.0 as **Core, Workshop, and Explorer publication mode** carrying:

- the v0.1 operator spine (initialise → author → snapshot → export → Explorer → copy brief),
- Phase 1 ISM Source Library (read-only ISM catalogue, no network egress at runtime),
- Phase 2 Requirement ↔ ISM Control mapping with rationale defaulted to sensitive,
- Phase 3 mapping confidence, lastReviewedAt, reviewBy plus ISM drift detection,
- the v0.5 Directions overlay (DIR-* entities, addressed-by/targets relationships, posture directionCount, deterministic Action Impact ranking),
- Workshop authoring of all v1.0 key fields including Directions and Action urgency hints,
- Australian context amplification (AU English, Home Affairs framing, local-friendly dates),
- the v0.4 UI resilience and readiness gates (writer-lock, backup/restore dry-run, explorer publication smoke, axe-core accessibility floor, ISM drift surfacing).

Performance reference for v1.0 is a **current MacBook Air**. The acceptance bar is "responsive on a fresh laptop", not a numeric throughput target.

## Out of scope for v1.0

The following are explicitly **deferred** past v1.0 and MUST NOT block the v1.0 release gates:

- Shop and Pub packages (`packages/shop` and `packages/pub`) — deferral notes only, no implementation.
- Explorer local-authoring mode (Explorer remains publication-only; ADR 0004 dual-mode local authoring stays deferred).
- Chart image export from Explorer or Workshop.
- Third-party accessibility audit. The axe-core floor in `scripts/check-accessibility.mjs` is the v1.0 ceiling; no external WCAG audit is commissioned.
- `plan-apply` import semantics. Imports stay at `full-replace` for v1.0.
- Editable posture (Workshop renders the posture brief; the operator cannot rewrite it in place).
- Numeric performance benchmarks beyond the "current MacBook Air responsiveness" feel test.

## Consequences

- The v0.5 slice (ADR 0023) is the last functional addition before v1.0 polish.
- `pspf-acceptance-and-quality-gates.md` v1.0 gates inherit v0.5 gates without adding chart export, local-authoring, or third-party a11y signals.
- Future versions (v1.1+) MAY introduce Shop, Pub, Explorer local-authoring, chart export, editable posture, plan-apply, and external a11y audit; each requires its own ADR.
- Release-readiness review SHOULD reference this ADR when deciding whether a candidate v1.0 build is "complete enough".
